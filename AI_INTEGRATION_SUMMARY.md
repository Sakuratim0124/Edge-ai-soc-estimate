## 📋 核心实现内容

### 1. 数据采集与缓冲管理
- **来源**: RTC中断每秒(可配置)触发 → ADC采样电压 → GPIO检测电流档位
- **缓冲**: 30个时间步 × 2个通道(电压/电流) 的环形缓冲区
- **存储**: 240字节 RAM

### 2. 智能推理触发机制
- **阈值**: 每收集20个数据点时执行一次推理
- **特点**: 不需要等待缓冲区满(30个)，20个就足以产生有效预测
- **滑动窗口**: 推理后删除前20个数据，新数据继续追加

### 3. 标准化处理 (Z-score)
| 通道 | Mean | Std Dev | 用途 |
|------|------|---------|------|
| 电压 | 3.6251 | 0.2615 | 将V_raw归一化 |
| 电流 | -1.9644 | 0.2645 | 将I_raw归一化 |
| SOC输出 | 52.535 | 28.375 | 反标准化模型输出 |

### 4. X-CUBE-AI模型接口
```c
// 新增公共函数 (app_x-cube-ai.c)
int ai_run_inference(float* input_data, float* output_soc)
```
- 输入: 60浮点数(30步×2通道)
- 输出: 1浮点数(原始预测的SOC)
- 集成方式: 直接调用X-CUBE-AI框架的 `ai_network_run()`

## 🔄 工作流程

```
每秒循环:
  1. RTC 中断
     └─ 调用 get_the_para()
        ├─ ADC 采样电压
        ├─ GPIO 检测电流档位
        ├─ 更新缓冲区 [count][0]=V, [count][1]=I
        ├─ count++, counter++
        └─ 若 counter >= 20:
           ├─ 调用 ai_inference_process()
           │  ├─ 标准化30个样本 → normalized_input[60]
           │  ├─ 调用 ai_run_inference()
           │  │  └─ X-CUBE-AI 推理 → raw_soc
           │  ├─ 反标准化: soc = raw_soc * 28.375 + 52.535
           │  ├─ 约束: 0 <= soc <= 100
           │  ├─ 设置 epd_update_flag = 1
           │  └─ 滑动缓冲区 (移除前20个,count→10)
           └─ counter 复位

  2. 主循环 (while)
     └─ 若 epd_update_flag == 1:
        └─ Update_BMS_UI(voltage, soc, soh)
           └─ 在电子纸上显示 SOC 百分比
```

## 💡 关键设计决策说明

### 为何用20个样本推理而非30个?
- 模型可接受较短序列
- 更快的推理周期 (20s vs 30s)
- 权衡准确度与响应速度

### 为何采用滑动窗口?
- 连续推理时保留部分历史
- 避免从零开始(可捕捉趋势)
- 内存占用恒定

### 为何分别标准化/反标准化?
- 机器学习模型期望标准化输入
- 输出需反标准化回原始量纲
- 两者参数由模型训练时决定
